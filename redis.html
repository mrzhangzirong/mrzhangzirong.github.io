为什么要用nosql？

1.单机mysql时代


图1

90年代，一个基本的网站访问量一般不会太大，单个数据库足够
那个时候，更多的是使用静态网页html，所以服务器没有太大压力
这时候网站的瓶颈是：

  • 数据量如果太大，单个数据库放不下
  • 数据的索引（B+Tree），一个机器内存也放不下
  • 访问量（读写混合），一个服务器承受不了

只要出现上面三种情况之一，必须升级

1.memcached+mysql+垂直拆分(读写分离)

  • 读写分离：
  • 数据库高速缓存：
    网站80%时间都在读 有时会查同一条数据，故在数据库间做一个缓存（数据库中间件），以减少压力。
    但是并不是一出来就有缓存的，
    发展过程：优化数据结构和索引->文件缓存（IO）->memcached(数据库高速缓存)

2.分库分表+水平拆分+mysql集群

  • 慢慢地就开始使用分库分表来解决表的压力！--做成mysql集群：

    本质：数据库（读、写）
    写的问题：
    MyISAM:表锁,十分影响效率！高并发下会出现十分严重的锁问题
    Innodb:行锁

3.最近的年代

  • 定位、音乐、热榜、浏览量10w+,mysql也有用它存储一些很大的文件，博客、图片，数据库表很大，效率就低了!
  • 因为数据量大而且变化很快，mysql等关系型数据库不够用了，需要有一种数据库来专门处理这种数据,减少mysql的压力
  • 大数据的io压力下，表几乎没法更改(1亿条加列)

5.为什么要用NoSQL

  • 用户的个人信息 ：社交网络，地理位置，用户自己产生的数据，用户的日志等等爆发式增长
  • nosql可以很好地处理以上的情况

什么是nosql？

  • nosql=not only sql
  • 泛指非关系型数据库，随着web2.0互联网的诞生，传统的关系型数据库很难对付web2.0时代，尤其是超大规模的高并发的社区，暴露出来很多难以克服的问题,nosql在当今大数据环境下发展得十分迅速，而且也是我们当下必须掌握的技术
  • 很多数据类型，例如：用户的个人信息，社交网络，地理位置等，这些数据类型的存储不需要一个固定的格式，不需要多么复杂的操作就可以横向扩展 Map(String,Object),使用键值对来控制

nosql特点

  1. 方便扩展（数据间没有关系，很好扩展）

  2. 大数据高性能（redis一秒写8万次，读11万，nosql的缓存记录是一种细粒度的缓存，性能会比较高）

  3. 数据类型是多样型的！（不需要事先设计数据库，随取随用，如果数据量十分大的表，很多人就无法设计了）

  4. 传统RDBMS或nosql

    • 传统的rdbms
      1. 结构化组织
      2. sql
      3. 数据和关系都存在单独的表中
      4. 操作数据定义语言
      5. 严格的一致性
      6. 寄出的事务
        ...
    • nosql
      1. 不仅仅是数据
      2. 没有固定的查询语言
      3. 键值对存储，列存储，文档存储，图形数据库（社交关系）
      4. 最终一致性
      5. CAP定理 和BASE（异地多活） 初级架构师！
      6. 高性能，高可用，高可扩
        ...

  5. 了解：大数据3v+3高
    大数据时代的3V：海量 多样 实时
    高并发 高可扩（随时可以水平拆分，机器不够了，可以扩展机器来解决） 高性能

  6. 真正在公司中 ：RDBMS+NOSQL才是最强的

  • redis是单线程的
  • redis是基于内存操作的，CPU不是redis的性能瓶颈，redis的瓶颈是机器的内存和网络带宽，既然可以使用单线程来实现，就是用了单线程
  • redis官方提供的数据是100000+QPS 完全不比Memecache差
  • redis为什么单线程还这么快？
    • 误区：高性能服务器一定是多线程吗？ 多线程一定比单线程效率高吗？
    • CPU>内存>硬盘
    • 核心：redis是把所有的数据都放在内存中了，所以说单线程操作是效率最高的，多线程(cpu上下文会切换，这操作会耗时) ，对于内存来说，如果没有上下文切换效率是最高的，多次读写都是在一个cpu上的，在内存情况下，这个是最佳的方案

五大数据类型

Redis 是一个开源（BSD许可）的，内存中的数据结构存储系统，它可以用作数据库、缓存和消息中间件。 它支持多种类型的数据结构，如 [字符串（strings）]， [散列（hashes）]， [列表（lists）]， [集合（sets）]， [有序集合（sorted sets）]与范围查询，，和 [地理空间（geospatial）]索引半径查询。 Redis 内置了 [复制（replication）]，[LUA脚本（Lua scripting）]， [LRU驱动事件（LRU eviction）]，[事务（transactions）]和不同级别的 [磁盘持久化（persistence）]， 并通过 [Redis哨兵（Sentinel）]和自动 [分区（Cluster）]提供高可用性（high availability）。
